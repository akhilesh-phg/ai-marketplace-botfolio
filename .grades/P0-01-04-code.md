## Findings (all non-blocking)

1. **DB round-trip test is not hermetic (highest priority).** `round-trip.test.ts`
   uses `describe.skipIf(!databaseUrl)` — it skips only when `DATABASE_URL` is *unset*.
   When the URL is set but the database is unreachable (no network, paused Supabase,
   CI without egress), the test **fails** instead of skipping (confirmed: `ENOTFOUND
   …pooler.supabase.com`). From P0-05 the CI `test` job is gate-blocking, so this can
   turn a network/infra hiccup into a red required check. Spec §P0-03 only mandates
   "skip … if unset", so this is technically compliant, but it's a latent CI fragility
   to flag for Opus (e.g. gate the live test behind an explicit `RUN_DB_TESTS` flag, or
   treat connection failure as skip).

2. **Branch-naming inconsistency.** `P0-01`/`P0-02` used `composer/P0-01`,
   `composer/P0-02`; `P0-03`/`P0-04` used `composer/p0-03-db`, `composer/p0-04-contracts`.
   The spec itself is self-inconsistent (§5 says `composer/P0-NN`; HANDOFF §6 says
   `composer/<slice-id-lowercased>` e.g. `composer/p0-03-db`), so this is partly a spec
   defect, but the executed branches drift across both styles.

3. **Commit-trailer drift.** `Model:`/`Spec:` trailers are inconsistent: P0-01/P0-02
   carry `Model:` but no `Spec:`; P0-03 carries `Spec:` but no `Model:`; only P0-04
   carries both (the AGENTS.md target). Harmless today (the `gate:no-new-decisions`
   machinery is built in P0-12), but worth tightening before gates go live.

4. **`identity()` placement.** Spec P0-01 step 9 implies an `identity.ts`; the function
   lives inline in `packages/shared/src/index.ts` (with `identity.test.ts` present).
   Cosmetic; fully covered.

5. **Health route duplicated in contracts.** `contracts/openapi.ts` instantiates an
   `OpenAPIHono` app *and a `/health` handler* purely to emit the OpenAPI doc; the real
   handler arrives in `apps/api` (P0-05). The spec's literal acceptance ("`buildOpenApi()`
   returns a doc with the health route registered") is met, but there are now two health
   handlers — watch that P0-05 derives its route from the contract rather than forking a
   second definition. `version` is hardcoded `'0.0.0'` here (env isn't available in
   contracts — acceptable).

6. **Bootstrap-stage coverage table is empty.** `vitest.config.ts` `coverage.include`
   targets `packages/core/**` (doesn't exist until P0-08) + `packages/shared/**`; the
   summary is correct (100% of shared) but per-file rows are blank. Expected at this
   stage; will populate at P0-08. Likewise `vitest.workspace.ts` lists `scripts` (created
   in P0-12) — currently tolerated by Vitest.
